# CSE 499B B-01 Provider Adapter Validation

Date: 2026-07-17  
Decision updated: 2026-07-20  
Requirement: B-01 permission-compliant public trail lookup  
Branch: `codex/b01-public-trail-adapter`  
Status: Historical rejected prototype; runtime code removed 2026-07-20

## Decision Update

The automated checks below prove that the experimental adapter normalizes and
labels provider responses safely. They do not prove that Nominatim reliably
finds the intended trail. The subsequent 24-trail evaluation found the intended
identity anywhere for 14/24 trails, ranked it first for 12/24, found 0/24 with
the current location-scoped form, and found no complete route relations.

Nominatim is therefore rejected as a supported B-01 source. The adapter, route,
and focused tests were removed on 2026-07-20 so the rejected provider cannot be
mistaken for a supported endpoint. This note and the reliability study preserve
the technical and product-decision evidence.

See
`docs/superpowers/validation/2026-07-17-cse-499b-nominatim-reliability.md`
for the authoritative/USGS/Nominatim comparison and final decision.

## Historical Experimental Scope

- Added a server-only Nominatim adapter and
  `GET /api/trailpack/trails/search` route.
- Normalized trail-like U.S. results into a bounded TrailPack response with
  OpenStreetMap identity, coordinates, optional bounding box and tags, source
  URL, attribution, retrieval status, confidence note, and explicit missing
  planning fields.
- Kept distance, elevation gain, route type, expected duration, and difficulty
  missing rather than trusting incomplete provider tags.
- Added controlled outcomes for invalid input, no results, ambiguous results,
  provider throttling, provider errors, timeouts, malformed payloads, and
  malformed object facts. Every outcome preserves manual fallback.
- Added an identifying user agent, HTTPS-only configurable provider and contact
  URLs, U.S. restriction, small result limit, response caching, and serialized
  one-request-per-second behavior within a warm server process.

## Open Gates

The provider failed the primary reliability gate independently of its technical
implementation. Passing normalization, fallback, and error tests cannot make a
50% first-result identity rate acceptable for a supported user-facing search.

The current request queue is shared by calls handled inside one warm server
process. Separate Vercel instances do not share module memory, so it does not
prove application-wide compliance with the public service's maximum of one
request per second. This remains additional evidence against deployment, but
solving it would not reverse the reliability rejection.

The replacement bounded Grand Teton workflow is recorded in
`docs/superpowers/validation/2026-07-20-cse-499b-grand-teton-public-source-import.md`.
It uses verified NPS values, reconciled USGS geometry, and manual no-result
fallback without a runtime Nominatim dependency.

## Disposition

1. Keep this document as negative feasibility evidence.
2. Do not restore the Nominatim route without a new source and reliability
   decision.
3. Continue Tetons-first expansion through individually verified NPS/USGS import
   records and preserve manual entry for trails that have not been approved.
